Nested Component (การสร้างComponentในComponentเพื่อให้ดึงComponentมาใช้เพื่อให้ทำให้จัดการข้อมูลง่าย)
เป็นสร้างComponentที่การเขียนComponentนั้นสามารถเขียนซ้อนกันได้คือการเขียนComponentหลายๆตัวภายใต้Componentอันเดียว
เพราะถ้าเอามาเขียนที่ๆเดียวจะไม่ค่อยเป็นระเบียบเท่าไหร่ การเขียนแบบนี้จะเป็นระเบียบขึ้นเพราะแบ่งเป็น Component Component
เช่นสร้าง Component หัวข้อ Componentรายละเอียดโปรแกรม Componentรายการธุรกรรมต่างๆ 
เพื่อให้เข้าไปจัดการแก้ไขเป็นส่วนๆได้ โดยที่ไม่ต้องเข้าไปแก้ที่จุดๆเดียว โดยมี Component App เรียกใช้เป็นก้อนๆ
