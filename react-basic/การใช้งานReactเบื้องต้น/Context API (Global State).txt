Context API (Global State)
นำมาใช้ตอนที่อยากให้ข้อมูลแอพของเรามีข้อมูลกลางเพื่อเอาข้อมูลกลางแชร์ไปยังคอมโพเน้นต่างๆ 
เพื่อให้คอมโพเน้นเข้าถึงข้อมูลชุเนั้นๆได้โดยที่ไม่ต้องมาคอยส่งข้อมูลไปมาจาก คอมโพแนนแม่ ไปยัง คอมโพแนนลูกที่ใช้เทคนิคการprops
แต่วิธีนี้ที่เอามาเช็คตลอดเวลา โดยจะสร้างกลางไว้ตัวนึงให้ไปกระจายเอาไว้ชุดนึง
โดยเราจะใช้ส่วนที่เป็น Provider (parent) กับ consumer (Children)
การเทียบง่ายๆคือ เปรียบเหมือน ป้ายเมนูอาหาร(Component) แม่ค้า(Consumer) พ่อครัว(Provider) เมนูอาหาร(ข้อมูล)
แม่ค้าจะยังไม่รู้ว่าเมนูอาหารมีอะไรบ้าง แต่พ่อครัวจะจัดการกับอาหารทำอาหารต่างๆขึ้นมา แล้วพอเสร็จแม่ค้าก้จะรู้ว่ามีเมนูอะไรบ้างแล้วไปเขียนป้ายเมนูอาหาร
